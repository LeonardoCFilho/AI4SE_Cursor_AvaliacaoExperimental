# Requisitos do Sistema de Reserva de Hotel

## Requisitos Funcionais (RF)

### RF-01 – Gestão de Quartos

| ID | Prioridade | Descrição |
|----|------------|-----------|
| RF-01.1 | M | O sistema deve permitir cadastrar quarto com número, capacidade, tipo (Básico, Moderno, Luxo), preço por diária e comodidades (Frigobar, café da manhã incluso, ar-condicionado, TV). |
| RF-01.2 | S | O sistema deve permitir cadastrar camas dos quartos, informando tipo (Solteiro, Casal King, Casal Queen). |
| RF-01.3 | M | O sistema deve listar quartos com as colunas: número, tipo, preço por diária e disponibilidade (Ocupado, Livre, Manutenção e Limpeza). |
| RF-01.4 | S | O sistema deve permitir editar quartos cadastrados a partir de um botão de edição na lista. |
| RF-01.5 | S | O sistema deve permitir alterar o status de disponibilidade do quarto (Ocupado, Livre, Manutenção e Limpeza). |

### RF-02 – Gestão de Hóspedes

| ID | Prioridade | Descrição |
|----|------------|-----------|
| RF-02.1 | M | O sistema deve permitir cadastrar hóspede com nome, sobrenome, CPF e e-mail. |
| RF-02.2 | M | O sistema deve listar hóspedes com colunas: nome, sobrenome e CPF (sem e-mail). |
| RF-02.3 | S | O sistema deve permitir editar hóspedes cadastrados. |

### RF-03 – Gestão de Reservas

| ID | Prioridade | Descrição |
|----|------------|-----------|
| RF-03.1 | M | O sistema deve listar reservas com número do quarto, tipo do quarto, nome do hóspede e disponibilidade em formato de chip. |
| RF-03.2 | M | O sistema deve permitir criar reservas associando quarto e hóspede. |
| RF-03.3 | S | O sistema deve permitir editar reservas existentes por meio de um botão de edição. |
| RF-03.4 | M | O sistema deve exibir disponibilidade do quarto na visualização de reservas (Ocupado, Livre, Manutenção e Limpeza). |

### RF-04 – Regras de Negócio

| ID | Prioridade | Descrição |
|----|------------|-----------|
| RF-04.1 | M | O sistema deve impedir reserva de quarto ocupado ou em manutenção/limpeza. |
| RF-04.2 | M | O sistema deve atualizar a disponibilidade do quarto ao criar ou cancelar/alterar reserva. |
| RF-04.3 | S | O sistema deve validar CPF (formato e unicidade). |
| RF-04.4 | S | O sistema deve validar e-mail. |
| RF-04.5 | M | O sistema deve garantir unicidade do número do quarto. |
